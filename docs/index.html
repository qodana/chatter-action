AI attribution for a normal Git workflow

Who wrote
this line?

Chatter shows which changes came from AI in a pull request, in a pipeline, and in a local clone. No separate Git workflow required.

Add it to a repository How it works ↓

Your code stays your code. Chatter only adds Git notes with attribution.

01

Add a workflow
to your repository

Create .github/workflows/chatter.yml. It runs for pull requests and after a merge. Chatter needs full Git history to match changes correctly.

.github/workflows/chatter.yml 
name: Chatter attribution

on:
  pull_request:
    types: [opened, synchronize, reopened, closed]

permissions:
  checks: write
  contents: write
  pull-requests: write

jobs:
  attribution:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - uses: qodana/chatter-action@main

This is the shortest setup. For a controlled production pin, replace main with a release SHA. If your repository restricts refs, allow the workflow to write refs/notes/*.

02

Install Chatter
on your computer

The installer puts the binary in ~/.chatter/bin and adds two Git hooks: one notices a new commit, the other publishes its note on push.

  1. 1

    Open a terminal in the root of the Git repository you want to track.

  2. 2

    Run this one command. It downloads and starts the installer.

terminal
curl -fsSL \
https://raw.githubusercontent.com/qodana/chatter-action/main/install.sh | sh

The installer never overwrites somebody else’s hooks. If post-commit or pre-push already belongs to another tool, it stops and tells you.

03

Then, keep working as usual

Your team does not need a new process.

  1. 01

    You make a commit

    A local hook records which lines are linked to an AI session.

  2. 02

    You push the branch

    Git notes with attribution travel with the branch.

  3. 03

    GitHub shows the result

    The action reads the notes and adds a clear report to the pull request.

  4. 04

    A merge keeps the history

    The action moves attribution to the final commit, including squash and rebase merges.

05

Get the data
in a local clone

There is no need to fetch notes by hand. Add --online and Chatter gets the attribution it needs from Git.

terminal
~/.chatter/bin/chatter blame src/example.js \
  --online --json --filter rollout --repo .

The JSON includes:

  • attributed lines;
  • the provider and model;
  • a chat link, when one exists;
  • lines without AI attribution as null.

A short checklist

If something is missing

  • The workflow has fetch-depth: 0.
  • The workflow has checks, contents, and pull-requests: write permissions.
  • Local hooks are installed in the same repository you push from.
  • Use blame --online for a local check.
